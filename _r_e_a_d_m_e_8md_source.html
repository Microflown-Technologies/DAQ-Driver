5 Using this library you can connect to a [Voyager](https://www.microflown.com/products/portable-measuring-systems/voyager/)
6 and use it as a DAQ ([Data Acquisition](https://en.wikipedia.org/wiki/Data_acquisition)) device.

8 ## Dependencies
9 - [CMake](https://github.com/Kitware/CMake) - CMake is a cross-platform, open-source build system generator.
10 - [IXWebsocket](https://github.com/machinezone/IXWebSocket) - Websocket library used to transport data as messages.
11 - [zeroconf](https://github.com/yvz/zeroconf) - mDNS library for automatic discovery of devices.
12 - [Protobuf](https://github.com/protocolbuffers/protobuf) - Google's language-neutral, platform-neutral, extensible mechanism for serializing structured data.
13 - [Qt Framework](https://github.com/qt) *(optional)* - Used by optional debugger GUI and provides extra functionality.

19 ## Building
20 There are also precompiled version of this library available through NuGet or via releases.
21 ### Windows
22 Make sure you have [Visual Studio 2019](https://visualstudio.microsoft.com/vs/) installed. Next, run the [configure.bat](configure.bat) script.
23 The script will download any missing dependencies and generate a Visual Studio Solution (SLN).
24 You can find in the SLN folder relative to the root of this project. You can include the generated solution in your own project.
25 
26 ### Linux
27 Make sure you have the required packages installed, preferably using your distro's package manager.
28 On Debian (or Debian based distro's like Ubuntu or Linux Mint) you can run this command to install all needed dependencies.
29 ```{r, engine='bash', count_lines}
30 sudo apt -y install build-essential cmake libprotoc-dev doxygen graphviz git protobuf-compiler
31 ```
32 Open a terminal and navigate to the root of this project and run the following commands.
33 ```{r, engine='bash', count_lines}
34 mkdir Build
35 cmake ..
36 make -j4
37 sudo make install
38 ```

40 ## Debugger
41 Optionaly you can also build the debugger. The debugger can be used to check the basic functionality of the DAQ driver.
42 To build the debugger you need Qt. You can [download Qt for free]((https://www.qt.io/download)).
43 Open the [CMakeList.txt](CMakeList.txt) file using Qt Creator.
44 Select a kit and go to the Projects tab, and change the 'BUILD_DEBUGGER' option from OFF to ON.
45 If you've configured your environment properly, it should now be possible to compile and run the project.
